# Telegram Shop Bot

Telegram Shop Bot — это учебно-портфолио проект Telegram-бота магазина на Python и aiogram 3.

Бот реализует полноценную логику интернет-магазина внутри Telegram: каталог товаров, карточки товаров, баланс пользователя, покупку товаров, историю покупок, админ-панель, управление товарами и финансовыми операциями.

Проект сделан с разделением логики по архитектуре:

```text
Handler → Service → Database
```

Это помогает держать код читаемым, расширяемым и удобным для поддержки.

## Установка и запуск локально

### 1. Клонировать репозиторий

```bash
git clone https://github.com/epifanovamur-cyber/telegram-shop-bot.git
cd telegram-shop-bot
```

### 2. Создать виртуальное окружение

Для Windows:

```bash
python -m venv .venv
.venv\Scripts\activate
```

Для Linux / macOS:

```bash
python3 -m venv .venv
source .venv/bin/activate
```

### 3. Установить зависимости

```bash
pip install -r requirements.txt
```

### 4. Создать `.env`

Создать файл `.env` на основе `.env.example`:

```env
BOT_TOKEN=your_bot_token_here
ADMIN_ID=your_telegram_id_here
```

### 5. Запустить бота

```bash
python bot.py
```

## Переменные окружения

Проект использует `.env` для хранения секретных данных.

Пример:

```env
BOT_TOKEN=your_bot_token_here
ADMIN_ID=your_telegram_id_here
```

Файл `.env` не загружается на GitHub, потому что содержит секретные данные.

В репозитории хранится только `.env.example`.

## Работа с базой данных

Проект использует SQLite.

В базе хранятся:

* пользователи;
* товары;
* покупки;
* балансы;
* финансовые операции;
* статусы товаров.

Файл базы данных `shop.db` не должен храниться в GitHub, потому что на сервере это живая база с реальными данными пользователей и покупок.

## Покупка товара и транзакции

Покупка товара выполняется через транзакцию.

Во время покупки бот проверяет:

* существует ли товар;
* активен ли товар;
* есть ли товар в наличии;
* хватает ли баланса пользователю;
* можно ли списать остаток;
* можно ли списать деньги;
* можно ли записать покупку;
* можно ли записать финансовую операцию.

Если на любом этапе возникает ошибка, выполняется rollback.

Это защищает от ситуаций, когда деньги списались, а товар не выдался, или товар списался, а покупка не сохранилась.

## Деплой

Проект подготовлен к деплою на VPS.

План деплоя:

```text
1. Купить VPS с Ubuntu
2. Подключиться по SSH
3. Установить Python, Git, venv и зависимости
4. Клонировать проект с GitHub
5. Создать .env на сервере
6. Установить requirements.txt
7. Проверить запуск через python bot.py
8. Настроить systemd
9. Запустить бота как службу
10. Проверить status и journalctl
```

После настройки systemd бот может работать на сервере 24/7.

